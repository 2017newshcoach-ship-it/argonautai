# Argonaut Blog Brand Guide — Tone & Manner + Output Rules (KO/EN) (v1.0)

이 문서는 Argonaut가 어떤 브랜드인지(정체성)를 글의 출력물에 고정하기 위한 **최상위 가이드**다.  
모든 글은 이 문서의 규칙을 먼저 만족해야 한다.  
(논증 설계/표/훈련 같은 “구현 방법”은 Writing Style Guide가 담당하며, 최종 출력은 반드시 Brand Guide 톤으로 번역되어야 한다.)

---

## 0) Purpose
Argonaut의 글은 “학원/튜터링”이 아니라 “전략가의 보고서”처럼 읽혀야 한다.

- 목표: 감정 설득이 아니라 **판정 기준(Decision Criteria)** 제공
- 인상: 차분, 단정, 정밀, 검증 가능
- 금지: 과장, 바이럴 톤, “꿀팁/핵팁/대박” 류의 감정형 카피

---

## 1) Brand Voice Definition
Argonaut는 다음을 지속적으로 반복해서 보여준다.

1) **정의(Definition)**: 지금 다루는 대상이 무엇인지  
2) **조건(Condition)**: 언제 이 판단이 성립하는지  
3) **예외/경계(Exception/Boundary)**: 언제 깨지는지  
4) **절차(Procedure)**: 무엇을 어떻게 확인/실행하는지  
5) **검증(Verification)**: 결과를 무엇으로 판단하는지(로그/지표/완료 조건)

---

## 2) Non-negotiable Rules (Must Follow)

### Rule A — 단정은 허용. 단, 조건을 붙인다(조건부 단정)
- 금지: “이건 항상 된다.”
- 허용: “(조건)이라면, 이 선택이 가장 안정적이다.”
- 필수: 최소 1회 이상 “깨지는 조건(반증/예외)”을 언급한다.

### Rule B — 좋다/나쁘다 대신 “기준 + 신호”로 말한다
- 금지: “좋다/나쁘다/핵심/중요”
- 허용: “(기준)에서 (오차/비용/시간)이 줄어든다.”
- 필수: 판단마다 **관찰 가능한 신호(signal/anchor)** 를 최소 1개 제시한다.

### Rule C — 절차는 번호로 쓰고, 완료 조건을 붙인다
- 실행 방법이 나오면 반드시 번호 목록으로 분해한다.
- 각 단계에는 “완료 조건(done condition)” 1개가 있어야 한다.
  - 예: “(지표)가 (기준)을 만족하면 다음 단계로 넘어간다.”

### Rule D — 불확실성은 숨기지 않는다(Explicit Uncertainty)
확인되지 않은 내용이 있다면 반드시 아래 3요소를 분리한다.
1) 현재 확인된 사실  
2) 아직 확인 전인 가설  
3) 확인 방법/시점(무엇을, 언제, 어떻게 검증할지)

- 금지: “아마/대충/느낌상/그냥”
- 허용: “현재 확인된 근거는 A다. B는 확인 전이며, (방법)으로 (시점)까지 검증한다.”

### Rule E — 전문적 거리(Professional Distance)
- 금지: 응원/격려형 마무리(“할 수 있어요”, “힘내요”)
- 금지: 감정적 결론(불안 조장/과도한 확신/선동)
- 권장 구조: **상황 → 원인 → 절차 → 검증**

### Rule F — 어휘 절제(Lexical Discipline)
- 꾸미는 형용사 줄이고, 기술어/정확한 단어를 늘린다.
- 긴 문장 1개보다 짧은 문장 2개를 우선한다.
- 문단은 2~4문장을 원칙으로 한다.

### Rule G — 독자 모드는 하나만(혼용 금지)
작성 전 반드시 하나를 선택하고, 끝까지 유지한다.

- **[MODE P] 학부모 모드**: “학부모님/어머님” 호명. 학생에게 직접 지시 금지. 관찰/점검/의사결정으로 번역한다.  
- **[MODE S] 학생 모드**: 학생에게 직접 말한다(너/여러분). 학부모 호명 금지.

### Rule H — 스캐폴드(내부 설계도) 노출 금지(Reader-first Output)
최종 글에 내부 작성 도구/라벨을 보이게 쓰지 않는다.

- 금지 예시:
  - “Proof Map”, “1단/2단/3단”
  - “Claim/Evidence/Warrant” 라벨
  - “Gate A/B/C” 같은 품질 게이트 용어를 본문/표 제목에 사용
- 허용:
  - 동일한 내용을 독자용 소제목으로 번역
  - 예: “판정 기준”, “왜 이 판단이 성립하는가”, “깨지는 조건”, “바로 확인하는 방법”

### Rule I — 외부 주장(정책/트렌드/뉴스) 위생(External-claim Hygiene)
정책/트렌드/뉴스를 “근거”로 쓰지 않는다.  
필요할 때만 아래 조건을 모두 만족해야 한다.

1) **구체성**: 누가/언제/무엇을 발표했는지  
2) **출처**: 링크/출처 제공  
3) **조건부 서술**: “이 사실이 의미하는 것 / 의미하지 않는 것” 분리

- 금지: “요즘은 다들…”, “최근 트렌드가…”
- 권장: 문항 내부 신호(anchor) + 재현 가능한 관찰로 대체

### Rule J — 은유(비유) 절제(Metaphor Discipline)
- 섹션당 은유 1개 이하(가능하면 0개)
- 금지: 전쟁/무기/전장, 독/블랙홀/파산/자판기/바이럴 쇼크 프레이밍
- 대체: 경계선(boundary), 비용(cost), 오차(error mode), 실패 조건(failure condition), 검증(verification)

## 3) Output Formatting Rules (KO)
한국어 출력에서 반드시 지킨다.

- 강한 줄바꿈: 핵심 전개는 1~2문장마다 줄바꿈
- 전환어는 논증 연결로 사용: “즉/하지만/정리하면/따라서”
- “입증 요구 질문”을 전체 글에 2~4회 포함(감정 질문 금지)
  - 예: “근거는 어디입니까?”
  - 예: “그 기준을 만족했습니까?”
  - 예: “이 판단이 깨지는 조건은 무엇입니까?”
  - 예: “지금 확인해야 할 신호(Anchor)는 무엇입니까?”

## 4) Preferred / Avoid Lexicon

### Preferred
decision, criterion, condition, branch, exception, boundary, error, error mode, verification, reproducibility, stability, consistency, evidence, variable, time limit, routine, record/log, failure condition

한국어 권장: 판단, 기준, 조건, 분기, 예외, 경계선, 오차, 오차 모드, 검증, 재현, 안정적, 일관성, 근거, 변수, 제한 시간, 루틴, 기록, 실패 조건

### Avoid
always, perfect, guaranteed, ultimate hack/tip, hype adjectives, emotional slogans, vague hedging, war/violence metaphors, poison/doom metaphors

한국어 금지: 무조건, 완벽, 보장, 꿀팁/핵꿀, 대박, 레전드, 미쳤다, 인생, 감정적 결론, 출처 없는 “요즘/최근”

## 5) Templates (Optional / Few-shot Seeds)
복붙 금지. 형태만 참고한다.

### Template A — Definition & Boundary
- “이 글에서 말하는 (개념)은 (정의)다.”
- “다만 (조건)이면 이 규칙은 깨진다.”

### Template B — If–Then Judgement
- “(신호/조건)을 확인한다.”
- “(조건)이라면, (판정)이 가장 안정적이다.”

### Template C — Procedure + Done Condition
1) “(행동)을 한다.”  
   - 완료 조건: “(지표)가 (기준)을 만족한다.”
2) “(다음 행동)을 한다.”  
   - 완료 조건: “(검증 결과)가 (기준)에 들어온다.”

### Template D — Uncertainty Handling
- “현재 확인된 사실: (A).”
- “확인 전인 가설: (B).”
- “확인 방법/시점: (언제) (무엇으로) 검증한다.”

## 6) Brand Compliance Checklist (Yes/No)
최종 글 제출 전 반드시 체크한다.

- 독자 모드([MODE P]/[MODE S])를 끝까지 유지했는가?
- 단정 문장에 조건을 붙였는가(조건부 단정)?
- “좋다/나쁘다” 대신 기준+신호로 설명했는가?
- 실행 방법은 번호 목록이며, 각 단계에 완료 조건이 있는가?
- 불확실한 내용은 확인/미확인/검증 방법으로 분리했는가?
- 외부 주장(정책/트렌드)을 근거로 쓰지 않았는가(예외 시 구체+출처+조건부)?
- 감정적 결론/격려 문장을 배제했는가?
- 스캐폴드(Proof Map/Claim-Evidence-Warrant/1단-3단)를 노출하지 않았는가?
- 은유를 절제했고 금지 은유를 쓰지 않았는가?
- 문단 길이(2~4문장)와 줄바꿈 규칙을 지켰는가?
